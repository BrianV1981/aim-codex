# A.I.M. Workflow Instructions (Continuity Standard)

This document is for **any agent** that enters this workspace.

## Your Core Duty
You are the guardian of the Operator's context. Your primary job is to ensure that when this session ends, the **next** session can pick up exactly where you left off.

## Memory Source of Truth
Use the core system:
- **Curated Long-Term:** `core/MEMORY.md` (Durable facts)
- **Daily Logs:** `memory/YYYY-MM-DD.md` (Narrative history)
- **Context Pulse:** `continuity/YYYY-MM-DD_HHMM.md` (Recent momentum)

## Standard Operating Procedure
1. **Startup:** Read `GEMINI.md` (root), then `core/IDENTITY.md`, `core/USER.md`, `core/AGENTS.md`, and `core/MEMORY.md`.
2. **Current State:** Check `docs/CURRENT_STATE.md` and the latest pulse in `continuity/` for the current status.
3. **Updates:** Every time a major decision is made, update `core/MEMORY.md` or `docs/DECISIONS.md`.
4. **Shutdown & Handoff:**
    - The **Handoff Protocol** is fully automated via the `SessionEnd` hook. Upon exiting, the system will automatically summarize the session, trigger the Flash Distiller, and create a new Context Pulse.
    - (Optional) Use `aim handoff` to create a manual, high-fidelity checkpoint during long sessions.

## Source Control (GitHub) Protocol
When instructed to commit and push changes, A.I.M. must use the automated versioning script rather than standard git commands.
- **Command:** `aim push "<Commit Message>"`
- **Why:** This script automatically stages all changes, appends a unique semantic version timestamp (`v1.YYYYMMDD.HHMM`), and pushes to `origin main`. This ensures the Sovereign Context Layer is perfectly traceable.
- **Rule:** Never use a raw `git push` unless specifically bypassing the versioning system.

## Handoff Protocol (Context Pulse)
The Context Pulse is a versioned file in `continuity/YYYY-MM-DD_HHMM.md`. It is automatically generated by the Distiller to capture the current mental model, accomplishments, and the "Edge". You may manually trigger a pulse generation at any time by executing `aim handoff`.

## Rules
- The chat history is transient; the files in `core/` are permanent.
- If it isn't in the root memory system, it didn't happen.
- Always assume the next agent is "Fresh" and needs the "Edge" defined clearly.
- Use the `hooks/` folder to maintain and expand this automation.
